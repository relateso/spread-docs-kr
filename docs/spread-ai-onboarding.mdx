## 환영합니다 👋

**모두의 창업을 통해 Spread AI에 신청해 주셔서 감사합니다.**

Spread AI는 한 문장으로 이렇게 요약됩니다.

> **AI가 영업을 실행합니다. 목적만 주면 타겟·시점·내용을 AI가 설계하고 실행합니다.**

이 가이드는 신청자분들이 가입 직후부터 첫 **AI Plan**을 만들어 실제로 이메일이 나가고, 반응을 읽어 다음 액션까지 자동으로 이어지도록 돕습니다. 순서대로 따라오시면 됩니다.

<Note>
  진행 중 막히는 부분이 있으면 언제든 인앱 채팅 또는 [we@spread.so](mailto:we@spread.so)로 문의해 주세요. 모두의 창업 신청자분들은 온보딩을 우선 지원해 드립니다.
</Note>

## Spread AI는 무엇을 자동화하나요?

기존 이메일 툴은 "누구에게, 언제, 무슨 내용을 보낼지"를 사람이 매번 정해야 했습니다. Spread AI는 이 판단을 **하나의 목표 단위(AI Plan)** 로 위임받아 대신 실행합니다.

- **설계**: 제품·타겟 정보를 주면 AI가 대상, 메시지, 발송 순서, 유도 행동(CTA)을 초안으로 잡습니다.
- **실행**: 조건에 맞는 연락처에게 이메일을 발송합니다.
- **반응 판단**: 열람·클릭·회신 같은 반응을 읽고, 지금 미팅을 제안할지 후속 메시지를 이어갈지 판단합니다.
- **통제**: 모든 자동화는 사용자가 정한 수준(확인만 받고 발송 / 알아서 발송) 안에서만 동작합니다.

## 1단계. 가입하면 AI 온보딩 채팅이 시작됩니다

Spread AI는 별도 설정 화면을 헤매지 않도록, **가입 직후 AI 채팅이 온보딩을 직접 안내**합니다. 채팅과 대화하다 보면 첫 캠페인 셋업이 끝나 있습니다.

<Tip>
  이메일·비밀번호 외에 **`Google로 가입`(로그인)** 으로도 간편하게 시작할 수 있습니다.
</Tip>

<Steps>
  <Step title="제품 분석">
    AI가 회원가입에 사용한 이메일 도메인(또는 입력한 웹사이트 URL)을 분석해 **제품 개요 · 핵심 기능 · 타겟 고객 · 가치 제안** 4가지를 초안으로 채워서 보여줍니다. 틀린 부분만 고쳐주면 됩니다.
  </Step>
  <Step title="목표(플랜) 생성 & 초안 작성">
    확인된 정보를 바탕으로 AI가 첫 **캠페인 목표(AI Plan)** 를 만들고, 대상·메시지·발송 순서 시나리오와 **첫 아웃리치 이메일 초안**까지 작성합니다.
  </Step>
  <Step title="연락처 추가">
    타겟 조건에 맞는 연락처를 찾아 플랜에 추가합니다. 아직 연락처가 없다면 엑셀/CSV 임포트로 채울 수 있습니다. (연락처가 없으면 실제 발송은 잠시 보류됩니다.)
  </Step>
  <Step title="초안 검토 & 테스트 발송">
    작성된 이메일 초안을 검토하고, 원하면 나에게 테스트 메일을 보내 실제 도착 모습을 확인한 뒤 시작합니다.
  </Step>
</Steps>

<Tip>
  온보딩 채팅은 이미지·자료를 붙여넣어도 이해합니다. 제품 소개서나 랜딩 페이지 캡처를 함께 주면 초안 품질이 올라갑니다.
</Tip>

### 먼저, 발송용 이메일을 연결하세요

AI가 실제로 메일을 보내려면 **발송에 사용할 이메일(도메인)을 먼저 연결**하는 것을 권장합니다. 방법은 두 가지입니다.

- **보유한 커스텀 도메인 연결** — 예: `you@yourcompany.com` (브랜드가 그대로 노출되고 전달률이 가장 좋습니다.)
- **개인 Gmail 계정 연결** — 도메인이 없다면 쓰던 Gmail 계정을 연결해 그 주소로 발송할 수 있습니다.

<Card title="발송용 도메인 연결하기" icon="badge-check" href="/domain">
  발송 도메인을 추가하고 소유권(SPF·DKIM·DMARC)을 인증하는 방법을 알아보세요.
</Card>

<Warning>
  **아무 이메일도 연결하지 않으면, 발송은 스프레드 공용 도메인 `@spreadsend.com` 으로만 가능합니다.** 이 경우 받는 사람에게 내 브랜드가 아닌 `@spreadsend.com` 주소로 보이고, 공용 도메인 특성상 전달률(스팸함 회피)도 내 도메인·Gmail 연결 대비 불리할 수 있습니다. 가능하면 커스텀 도메인이나 개인 Gmail 계정을 연결한 뒤 발송하시길 권장합니다.
</Warning>

## 2단계. 캠페인 목표(AI Plan) 이해하기

**캠페인 목표(AI Plan)는 Spread AI의 핵심 단위**입니다. 하나의 목표는 "어떤 아웃바운드를, 누구에게, 어떻게 실행할지"를 담고 있고, 대부분의 설정은 **AI 채팅에 말로 하면** 됩니다.

목표는 아래 6가지로 구성됩니다.

| 항목 | 의미 | 예시 |
| --- | --- | --- |
| 누구에게 | 보낼 대상 | "최근 데모를 신청한 SaaS 담당자" |
| 무슨 말을 | 핵심 메시지 | "온보딩 시간을 절반으로 줄여준다" |
| 어떤 채널로 | 보낼 채널 | 이메일 |
| 어떤 순서로 | 보내는 순서 | "첫 메일 → 3일 후 팔로업 → 7일 후 미팅 제안" |
| 어떤 유도 행동으로 (CTA) | 상대가 하길 바라는 행동 | "키워드 회신" → "미팅 링크" |
| 얼마나 자주 | 보내는 빈도 | "영업시간 내, 과하지 않게" |

목표 상세 화면은 **개요 · 연락처 · 지침(Overview · Contacts · Instruction)** 탭으로 나뉘고, 보내는 순서는 **발송 흐름(Flow)**, 답장·반응에 대응하는 방식은 **반응 시나리오(Scenario)** 로 관리됩니다.

<Tip>
  설정을 바꾸고 싶을 때는 화면에서 버튼을 찾기보다, 우측 **AI 채팅 창(AI 채팅 패널)** 에 "팔로업 간격을 3일에서 5일로 바꿔줘"처럼 말하는 것이 가장 빠릅니다. **원하는 게 복잡하거나 궁금한 점이 있으면 무엇이든 이 채팅으로 물어보세요.**
</Tip>

## 3단계. AI에게 어디까지 맡길지 정하기

Spread AI의 가장 중요한 개념입니다. AI에게 **어디까지 맡길지**를 두 가지로 조절합니다. 처음에는 보수적으로 시작해, 믿음이 쌓이면 점점 더 맡기는 것을 권장합니다.

### 발송을 얼마나 자동으로 할지 (실행 수준 / Execution tier)

<AccordionGroup>
  <Accordion title="끄기 (꺼짐 / Off)" icon="circle-pause">
    목표가 잠시 멈춘 상태입니다. 아무것도 발송하지 않습니다.
  </Accordion>
  <Accordion title="확인 후 발송 (확인 / Ask)" icon="circle-check">
    AI가 동작하되 **보내기 전에 매번 사용자에게 확인을 받습니다.** 처음 시작할 때 권장합니다.
  </Accordion>
  <Accordion title="알아서 발송 (자율 / Autonomous)" icon="wand-magic-sparkles">
    AI가 확인 없이 **알아서 발송**합니다. 흐름이 충분히 검증된 뒤 사용하세요.
  </Accordion>
</AccordionGroup>

### 받은 답장에 얼마나 자율적으로 대응할지 (답장 대응 수준 / 리액션 자율도)

<AccordionGroup>
  <Accordion title="정해진 것만 처리 (조건부 처리 / Strict)" icon="shield-halved">
    미리 정해둔 상황의 답장만 AI가 처리하고, **예상 밖 상황은 사람에게 알립니다.** 가장 안전한 설정입니다.
  </Accordion>
  <Accordion title="AI가 가이드 따라 처리 (LLM 보조 / Augmented)" icon="robot">
    AI가 **정해둔 시나리오를 참고해** 답장에 대응합니다.
  </Accordion>
  <Accordion title="AI가 알아서 처리 (LLM 자율 / Autonomous)" icon="brain-circuit">
    AI가 시나리오와 무관하게 **스스로 판단해** 답장에 대응합니다.
  </Accordion>
</AccordionGroup>

### 애매하면 사람에게 물어봅니다 (승인 요청 / 승인 게이트)

AI가 스스로 판단하기 애매한 경우(예: 정해둔 시나리오를 벗어난 상황, 보내기 전 확인이 필요한 경우)에는 실행을 멈추고 **승인 필요** 항목으로 남깁니다. 사용자는 각 건을 **승인 / 거절**하거나, "이 연락처는 앞으로 자동 허용"으로 처리할 수 있습니다.

<Note>
  즉, 자동화 수준을 높여도 사람이 완전히 빠지지 않습니다. AI는 애매한 순간마다 사용자에게 결정을 넘기도록 설계되어 있습니다.
</Note>

## 4단계. 반응을 읽고 다음 액션으로

플랜이 돌기 시작하면, 발송된 이메일과 그에 대한 반응이 자동으로 기록·관리됩니다.

<CardGroup cols={2}>
  <Card title="캠페인 메일 (AI Campaigns)" icon="paper-plane">
    AI가 여러 대상에게 보낸 캠페인 이메일을 모아 봅니다.
  </Card>
  <Card title="개별 메일 (AI Emails)" icon="envelope">
    개별 상황에 맞춰 AI가 작성·발송한 이메일을 모아 봅니다.
  </Card>
</CardGroup>

관심이 확인된 리드는 즉시 미팅 제안으로, 아직 이른 리드는 후속 관리로 — 상대에게 유도하는 행동(CTA)은 상황에 따라 "키워드 회신"에서 "미팅 링크"로 자연스럽게 올라갑니다.

## 5단계. 성과 확인하기 (AI 리포트)

**리포트** 메뉴에서 플랜별 이메일 활동을 **주간 / 일간**으로 확인할 수 있습니다.

추적되는 지표: **발송 · 답장 · 응답률 · 열람 · 열람률 · 클릭 · 반송 · 신규 컨택**

<Note>
  해당 기간에 활동이 있는 플랜만 리포트에 표시됩니다. 아직 발송이 없으면 비어 있을 수 있습니다.
</Note>
